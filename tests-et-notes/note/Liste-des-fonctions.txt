LISTE DE QUELQUES FONCTIONS NECESSAIRES
N.B. : II peut y avoir des erreurs/oublies (dans l’Endpoint, dans les Params ou dans les
Retour), ou des fonctions à développer avant de développer les fonctions ci-dessous
dans leur Service respective mais il faut toujours se référer aux règles de gestion pour
savoir quoi faire lors de leur développement.




1 Authentification & Profil – AuthService

registerUser
        •      Endpoint : POST /api/auth/register
        •      Params : { user : User }

        •      Retour : { user: User, token: string, portefeuille: PorteFeuille }

        •       Description : Inscription Commercant ou Acheteur. Création automatique du
    PorteFeuille.

login
        •      Endpoint : POST /api/auth/login
        •      Params : { email: string, mdp: string }
        •      Retour : { user: User, token: string }
        •      Description : Authentifie Admin / Commercant / Acheteur


getMyProfile


        •      Endpoint : GET /api/users/:id/me
        •      Params: { id : any }
        •      Retour : User
        •      Description : Récupère le profil de l’utilisateur connecté

updateMyProfile
        •      Endpoint : PUT /api/users/me
        •      Params : { userId : any, data : User }
        •      Retour : User (déjà à jour)
        •      Description : Mise à jour des infos personnelles


2 Notifications – NotificationService

getMyNotifications
        •      Endpoint : GET /api/users/:userId/notifications
          •      Query params : { userId : any, page: number, limit: number }
          •      Retour : { data: Notification[ ], total: number, unreadCount: number }
          •      Description : Liste paginée des notifications utilisateur

markNotificationAsRead
          •      Endpoint : PUT /api/notifications/:id/read
          •      Params : { id : any }
          •      Retour : Notification
          •      Description : Marque une notification comme lue


createNotification (interne non exposé aux api)


       •       Params : { type: NotificationTypeEnum, message: string, receveur: UserId,
   urlRoute?: string }
          •      Description : Création auto lors d’un achat, paiement, vente


3 PorteFeuille – PorteFeuilleService

getMyWallet
          •      Endpoint : GET /api/users/:id/wallet
          •      Params : { id: any }
          •      Retour : { wallet: PorteFeuille, transactions: PFTransaction[ ] }


createTransaction (interne non exposé aux api)


      •      Params : { fromWallet: PorteFeuille, toWallet: PorteFeuille, type:
   PFTransactionTypeEnum, amount: number, description?: string }
          •      Description : Gère Achat / Loyer / Commission. Met à jour les balances.
